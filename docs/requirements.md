# 📋 需求文档 —— 知路 ZhiPath

## 一、项目背景

当前大学生学习面临三大痛点：

1. **难以自我诊断**：学生不清楚自己对哪些知识点掌握薄弱，往往在考试中才暴露问题
2. **学习路径模糊**：面对复杂的知识体系，不知道应该先学什么、后学什么
3. **缺乏个性化指导**：传统教学一刀切，无法针对每个学生的薄弱点提供定制化帮助

**知路 ZhiPath** 旨在通过知识图谱、自适应诊断和AI辅导，构建 **诊断 → 可视化 → 规划 → 辅导 → 复测** 的完整学习闭环。

## 二、目标用户

- **核心用户**：北航本科生，尤其是 C 语言程序设计、高等数学等基础课程的学生
- **扩展用户**：任何需要系统化学习和自我诊断的大学生

## 三、核心功能

### 3.1 知识图谱可视化
- 以力导向图展示课程知识点及其依赖关系
- 节点颜色反映用户掌握程度（红/黄/绿三色灯）
- 点击节点查看详细信息和学习建议

### 3.2 智能诊断测评
- 基于 BKT（贝叶斯知识追踪）算法的自适应测评
- 优先诊断未测试和疑似薄弱的知识点
- 输出各知识点掌握概率和薄弱点列表

### 3.3 个性化学习路径
- 基于拓扑排序和薄弱度优先的路径规划
- 自动纳入未掌握的前置依赖
- 展示预计学习时间和进度

### 3.4 AI 苏格拉底式答疑
- 默认引导模式：通过提问帮助学生自己发现答案
- 可选直答模式：直接给出详细解释
- RAG 检索增强，基于教材内容回答

### 3.5 学习报告
- 做题统计、正确率、活跃天数
- 掌握度雷达图
- 进步趋势曲线

## 四、用户故事

| # | 角色 | 故事 | 优先级 |
|---|------|------|--------|
| 1 | 学生 | 我想快速了解自己C语言哪些知识点薄弱 | P0 |
| 2 | 学生 | 我想看到知识点之间的关系，知道先学什么 | P0 |
| 3 | 学生 | 我想获得一条适合我的学习路径 | P0 |
| 4 | 学生 | 做错题时，我想得到引导而不是直接答案 | P1 |

## 五、非功能性需求

- **性能**：API 响应时间 < 500ms，知识图谱渲染 < 2s
- **安全**：JWT 认证，密码 bcrypt 加密存储
- **可用性**：响应式设计，支持桌面和移动端浏览器
- **可扩展性**：支持新增课程，只需添加知识图谱 JSON 文件

## 六、技术约束

- 前端：React 18 + Vite + Ant Design
- 后端：Python 3.11 + FastAPI
- 数据库：SQLite (开发) / MySQL (生产)
- 图数据库：Neo4j 5（可降级为 JSON 文件后备）
- AI：OpenAI API + LangChain + FAISS
